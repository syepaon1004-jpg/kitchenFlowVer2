## 0. 프로젝트 현재 상태 요약

### 0.1 완료된 것

**패널 시스템 (1차)** — 완전 교체 완료:
- 기존 파노라마 이미지 + SVG 히트박스 → CSS 3D 종이접기 패널 시스템으로 전면 교체
- 편집 모드: 배경 이미지 위 패널 3장(평면), 장비 7종(서랍/폴드냉장고/바구니/화구/씽크대/작업대/선반) 배치
- 미리보기/인게임: CSS 3D로 패널 접기 → 주방 단면도 형성, 장비 인터랙션(서랍 열림, 화구 불 단계, 바구니 펼침, 폴드냉장고 열림 등)

**재료·그릇 배치 시스템** — 완전 완료:
- 패널 표면에 재료/그릇 사각형 자유 배치 (FK 연결)
- 서랍/바구니 내부 N×M 그리드 편집 (셀 합치기/나누기/재료 FK 연결)
- 폴드냉장고 내부 자유 배치 편집 (2단계 깊이: 냉장고 → 내부 바구니 → 그리드)
- 미리보기/인게임에서 모든 내부 아이템 3D 렌더링

**묶기(bind) 기능** — 완료:
- 바구니 셀 묶기로 2층 적재 구조 표현
- 묶인 셀은 앵커 셀의 bottom을 공유 회전축으로 사용

**기존 게임 로직** — 미연결 상태로 존재:
- equipmentStore: 웍 물리(온도/stir/overheating), 씽크대 세척
- gameStore: 재료 인스턴스, 그릇 인스턴스, 주문
- uiStore: 모달(OrderSelectModal, QuantityInputModal)
- 이 로직들은 @dnd-kit 기반으로 트리거됨 → 클릭/선택 방식으로 트리거 변경 필요

## 1. 인터랙션 모델

### 1.1 핵심 개념: "선택 → 대상 클릭 → 액션"

모든 인게임 상호작용은 2단계로 이루어진다:
1. 오브젝트를 클릭하여 "현재 선택"에 등록
2. 대상을 클릭하여 액션 발생

"현재 선택"은 HUD에 표시되어 사용자가 현재 무엇을 들고 있는지 항상 인지할 수 있다.

### 1.2 선택 가능한 오브젝트

| 선택 대상 | 클릭 위치 | 선택 후 표시 |
|-----------|----------|------------|
| 재료 | 서랍 셀, 바구니 셀, 냉장고 내부 재료, 냉장고 내부 바구니 셀 | 재료명 + 아이콘 |
| 그릇 | 패널 표면 그릇 사각형 | 그릇명 |
| 올려놓인 그릇 | 패널 2 위에 올려놓인 그릇 | 그릇명 + 내용물 |
| 웍 내용물 | 홀로그램(=웍 UI) | 웍 내용물 목록 |

**핸드바**: Phase 3에서 설계서와 다르게 구현됨. 핸드바 전체 선택(type='handbar')은 제거. 핸드바 내부 개별 재료 칩을 클릭하여 ingredient로 선택 (instanceId 포함).

### 1.3 액션 매트릭스

| 현재 선택 | 클릭 대상 | 액션 |
|-----------|----------|------|
| 없음 | 재료 소스 | 재료를 "현재 선택"에 등록 |
| 없음 | 그릇 소스 | 그릇을 "현재 선택"에 등록 |
| 없음 | 홀로그램(=웍) | 웍 내용물을 "현재 선택"에 등록 (내용물 유무 무관) |
| 없음 | 올려놓인 그릇 | 해당 그릇을 "현재 선택"에 등록 |
| **재료** | 홀로그램(=웍) | 재료 투입 (단위별 분기) |
| **재료** | 올려놓인 그릇 | 재료 투입 (단위별 분기) |
| **재료** | 핸드바 | 재료를 핸드바에 보관 |
| **그릇(소스)** | 작업대/화구(패널 2) | 클릭 위치에 그릇 올려놓기 (place-container) |
| **올려놓인 그릇** | 작업대/화구(패널 2) | 그릇 이동 (move-container) |
| **올려놓인 그릇** | 홀로그램(=웍) | 붓기 — 그릇 → 웍 (pour) |
| **올려놓인 그릇** | 올려놓인 그릇 | 붓기 — 그릇 → 그릇 (pour) |
| **올려놓인 그릇** | 씽크대 | 빈 그릇 생애주기 종료 (dispose) |
| **웍 내용물** | 올려놓인 그릇 | 붓기 — 웍 → 그릇 (pour) |
| 모든 선택 | 빈 영역 | 선택 해제 |
| 모든 선택 | 현재 선택된 오브젝트를 다시 클릭 | 선택 해제 |
| 모든 선택 | 현재 선택 HUD 영역 클릭 | 선택 해제 |

### 1.4 투입량 결정 (단위별 분기)

| unit | 동작 |
|------|------|
| spoon, portion, pinch, handful, ladle, spatula, **ea** | 즉시 투입 (qty=1). 같은 재료 재투입 시 기존 인스턴스에 누적 |
| g, ml | 투입량 선택 **버튼** 팝업 |

**투입량 팝업:**
- input 사용 금지 (모바일/패드 키보드 회피)
- 레시피 요구량 기준으로 버튼 생성
- 예시: 요구량 3 → [1, 2, 3] / 요구량 5 → [5, 10, 15] / 요구량 20 → [10, 20, 30]
- 기존 QuantityInputModal을 수정: input → 버튼 UI로 전환

### 1.5 핸드바

- 위치: HUD 좌측 상단, SelectionDisplay 위
- 재료를 임시 보관하는 공간
- 표시: 보관된 재료 칩 목록
- 재료 선택 → 핸드바 클릭 → 재료가 핸드바에 들어감
- 핸드바 재료 칩 직접 클릭 → 해당 재료를 ingredient로 선택 (instanceId 포함)
- 선택된 핸드바 재료 → 웍/그릇 클릭 → 분할 전달 (단위 분기)
- 핸드바→핸드바 루프 방지 (instanceId 존재 시 차단)
- DnD 공존: 기존 useDraggable 유지, PointerSensor activationConstraint: { distance: 5 }로 클릭/드래그 구분

## 3. 그릇 올려놓기 시스템

### 3.1 올려놓기 가능 장비

패널 2 위의 작업대(worktop)와 화구(burner)에만 올려놓기 가능.

작업대와 화구 모두 패널 2 전용 CHECK 제약 적용됨 (SQL 실행 완료).

### 3.2 올려놓기 메커니즘

그릇 소스를 선택한 상태에서 작업대 또는 화구를 클릭하면:
1. 클릭 좌표를 해당 장비 내 로컬 비율(0~1)로 변환
2. 해당 위치에 그릇 인스턴스 생성 (Zustand — 세션 중 DB write 금지)
3. 위치 저장: panel_equipment_id + 장비 내 로컬 좌표(x, y — 0~1)
4. 3D 렌더링: 패널 2 표면에서 `transformOrigin: bottom center` + `rotateX(-90deg)` → 수직으로 세워짐
5. 올려놓인 그릇은 클릭 가능 (재선택 → 이동/붓기)

왜 장비 내 로컬 좌표인가: "작업대/화구 위에만 올려놓을 수 있다"는 제약이 데이터 구조에 의해 자동 강제된다. 패널 기준 좌표였다면 매번 장비 경계 검증이 필요하고 오류 가능성이 커진다.

### 3.3 그릇 생애주기

```
그릇 소스(패널 표면 아이템) → 선택 → 올려놓기(작업대/화구 위)
  → 재료 투입 (여러 번)
  → 붓기(pour)로 내용물 수신/전달
  → (레시피 판별 → 서빙 — 후속 구현)
  
빈 그릇 → 씽크대 클릭 → 생애주기 종료 (disposed)
붓기 후 빈 웍 → dirty 상태 → 원래 상태로 복귀
```

### 3.4 올려놓인 그릇 렌더링

- 패널 2 위에 동적으로 생성되는 3D 사각형
- 크기: PLACED_CONTAINER_SIZE_VH (8vh) 정사각형 — 상수 관리
- 외형: 흰색 사각형 + 테두리 + 그릇명 텍스트 + 내용물 표시
- transform: `translate(-50%, -100%) rotateX(-90deg)`, transformOrigin: bottom center
- preserve-3d 체인에 포함 (scene → panel1 → panel2 → placed-container)
- pointer-events: none (hit-test는 scene 레벨 getBoundingClientRect로 수행)
- data-click-target="placed-container" + data-click-meta에 containerInstanceId 포함

## 4. 붓기(pour) 시스템

### 4.1 범용 설계 원칙

붓기는 **"내용물이 있는 용기 → 다른 용기로 내용물 전체 이동"**이라는 단일 동작이다.
소스와 목적지가 무엇인지만 달라지며, 동작 자체는 동일하다.

모든 내용물 이동은 pour 액션으로 통합한다. merge-containers를 별도 액션으로 분리하지 않는다.

### 4.2 pour 매트릭스

| source 선택 | destination 클릭 | 결과 |
|-------------|-----------------|------|
| wok-content | placed-container | 웍 재료 → 그릇 이동 + plate_order 부여 |
| placed-container | hologram(=웍) | 그릇 재료 → 웍 이동 |
| placed-container | placed-container (다른 그릇) | 그릇 재료 → 다른 그릇 이동 + plate_order 부여 |
| (미래) 튀김채 내용물 | placed-container | 튀김채 재료 → 그릇 이동 |
| (미래) placed-container | 튀김채 | 그릇 재료 → 튀김채 이동 |

### 4.3 pour 액션 구조

resolveAction이 반환하는 pour 액션:
- source 정보: 현재 선택에서 추출
  - wok-content 선택 → equipmentStateId (어느 웍인지)
  - placed-container 선택 → containerInstanceId (어느 그릇인지)
- destination 정보: add-ingredient와 동일한 destination 구조
  - `{ locationType: 'equipment', equipmentId }` — 웍으로 이동
  - `{ locationType: 'container', containerInstanceId }` — 그릇으로 이동

### 4.4 재료 이동 방식 (bulk)

pour 실행 시 source 용기의 모든 재료 인스턴스를 **set() 1번으로 한번에** 이동한다.
gameStore에 bulkMoveIngredients 액션을 추가한다.

각 재료의 변경:
- location_type → destination의 locationType
- equipment_state_id 또는 container_instance_id → destination의 FK (나머지 FK는 null)
- plate_order → destination이 container일 때 부여 (동일 pour의 모든 재료 = 같은 plate_order)

### 4.5 plate_order 규칙

- 같은 붓기에 이동하는 모든 재료가 동일한 plate_order를 받는다
- plate_order 값 = destination 그릇의 current_plate_order + 1
- incrementContainerPlateOrder 호출로 값 획득
- destination이 equipment(웍)일 때는 plate_order를 부여하지 않는다 (null)

### 4.6 wok-content 선택

- 선택 없음 + hologram 클릭 → 항상 select(wok-content) (내용물 유무 무관)
- 내용물 없는 웍을 선택한 후 pour 실행 시, 이동할 재료가 없으면 무시

### 4.7 붓기 후 처리

- source가 웍: dirty 상태 설정 (웍 전용 — equipmentStore 또는 game_equipment_state)
- source가 그릇: 빈 그릇 상태 유지 (disposed는 씽크대에서 별도 처리)
- 빈 웍(dirty) → 원래 상태로 복귀 (세척은 미래 구현)

## 5. 장비 연결

### 5.1 화구 ↔ 웍

- 불조절 버튼: 현재 로컬 fireLevel 순환 → equipmentStore의 burner_level에 연결
- 볶기 버튼: 현재 disabled → equipmentStore의 stir 누적에 연결 (홀드 시 매초 누적)
- 홀로그램: 현재 웍 내용물 텍스트 표시 (wokContentsMap 기반)
- 온도: equipmentStore의 wok_temp에 따라 화구 색상/상태 표시
- 선택 있을 때 burner 클릭: 불 순환 없이 click system으로 위임 (hasSelection prop 분기)

### 5.2 씽크대

- 빈 그릇 선택 → 씽크대 클릭 → 생애주기 종료 (dispose)
- 웍 세척: 미래 구현 (dirty 웍이 씽크대로 이동하는 시각적 표현)

### 5.3 서랍/바구니/냉장고

- 재료 소스 역할: 각 셀/아이템이 연결된 ingredientId로 재료 인스턴스를 생성
- 클릭 시 해당 재료를 "현재 선택"에 등록
- 무한 소스: 재료를 선택해도 소스가 소진되지 않음 (훈련 시스템이므로)

## 7. 새 상태 관리: 선택 시스템

### 7.1 선택 상태

```
selectionStore (Zustand):
  selection: SelectionState | null
  select(state): void
  deselect(): void
  reset(): void

SelectionState:
  type: 'ingredient' | 'container' | 'wok-content' | 'placed-container'
  
  ingredientId?: string          // ingredient: 어떤 재료인지
  instanceId?: string            // 유한 소스(핸드바) 구분
  sourceEquipmentId?: string     // 어느 장비에서 가져왔는지
  sourceLabel?: string           // 표시명
  
  containerId?: string           // container: 어떤 그릇인지
  
  containerInstanceId?: string   // placed-container: 올려놓인 그릇 인스턴스
  
  equipmentStateId?: string      // wok-content: 어느 웍인지
```

### 7.2 resolveAction 아키텍처

```
resolveAction(currentSelection, clickTarget) → ResolvedAction | null
```

- 순수 함수 (스토어 접근 없음)
- 현재 선택 상태 × 클릭 타겟 → 액션 결정
- 스토어 접근이 필요한 판단(웍 내용물 유무 등)은 호출 측에서 처리

### 7.3 onAction 위임 구조

useClickInteraction은 선택 로직만 담당. 비즈니스 액션(add-ingredient, pour, place-container, move-container, dispose 등)은 onAction 콜백으로 GamePage에 위임.

## 9. 변경 불가 원칙 (계속 유효)

- assigned_order_id 묶음 (group_id 금지)
- 재료 인스턴스는 **클릭 완료 후에만 생성** (선택 시 생성 금지)
- any 타입 금지 (unknown + 타입 가드)
- Zustand 셀렉터 인라인 filter/map 금지

## 10. 위험 요소

1. **hit-test 복잡도**: 서랍/바구니/냉장고 내부 셀은 3D 공간 내부 요소. 장비 열림 상태에서만 클릭 가능해야 함. 닫힌 서랍의 셀은 클릭 불가.
2. **상태 전이 매트릭스**: "현재 선택 × 클릭 대상" 조합이 많음. 누락된 케이스가 있으면 UX 혼란.
3. **올려놓인 그릇 렌더링**: 런타임 동적 3D 요소. preserve-3d 체인 + 패널 2 좌표 변환 필요.
4. **기존 Zustand 스토어 인터페이스**: DnD 전제로 설계된 인터페이스를 클릭 방식에 맞게 조정 필요.
